## 一、選對招聘平臺：Web3 求職的黃金法則

人們常說：“工欲善其事，必先利其器。” 想要找到一份稱心如意的 Web3 工作，除了熟人內推，選對招聘平臺是關鍵。而選擇平臺的標準其實不難，這裡有一個黃金法則供參考：“提供的招聘資訊是否及時、真實、準確”。

**“及時”** 指的是該崗位的 HC（Head Count，招聘名額）仍處於 Open 狀態。很多時候你看到的招聘資訊流可能已經是不知道經過幾手傳遞的末端訊息，顯然得不償失不是嗎？

**“真實”** 指的是該崗位是真實存在的，不存在憑空捏造的局面，有一些不良媒體為了引流，會惡意編造一些不存在的崗位。

**“準確”** 崗位描述、薪資範圍、工作地點、彙報物件、面試流程，甚至入職後期望交付的 KPI，都應與實際情況高度匹配。很多求職者上崗後的“落差感”往往正是因為資訊不準確。

在這樣的判斷標準下，有一些招聘平臺就略顯“野雞”了。在這裡推薦一些在這三個方面做得較好的平臺。

## 二、優質招聘平臺推薦

這裡的平臺都是滿足幾個特點：時間悠久（老牌）、真實與可靠
::: steps

1. [**Web3.career**](https://web3.career)
   - **簡介：** Web3.career 是一個專注於 Web3 領域的全球性招聘平臺，彙集了大量來自區塊鏈、加密貨幣、DeFi、NFT、元宇宙等領域的職位。其特點是職位更新及時，覆蓋面廣，且提供了詳細的職位描述和公司資訊，是 Web3 求職者的重要資源。
     <br>
2. [**DeJob**](https://www.dejob.top/)
   - **簡介：** DeJob 是一個專注於 Web3 和遠端招聘的平臺，旨在幫助 Web3 專案方和求職者進行匹配。該平臺支援免費釋出職位，且職位更新較快，尤其適合國內 Web3 專案和求職者。
     <br>
3. [**SmartDeer**](https://smartdeer.com/)
   - **簡介：** SmartDeer 是一個新興的 Web3 人才招聘平臺，專注於連線優質的 Web3 專案和專業人才。平臺提供精準的崗位匹配服務，涵蓋技術開發、產品運營、市場推廣等多個職能領域。其特點是注重人才質量和專案可靠性，為求職者提供更有保障的就業選擇。

:::

## 三、其他資訊渠道

除了專業的招聘平臺，以下渠道也是獲取 Web3 工作資訊的重要來源：

- **[LXDAO](https://lxdao.io)：** LXDAO 是一個由開發者、研究員、設計師等組成的去中心化自治組織（DAO），經常整理和釋出社群成員或合作專案的招聘資訊，同時提供各類 Bounty 崗位機會，尤其適合技術和研究背景的求職者。LXDAO 社群：[https://t.me/LXDAO](https://t.me/LXDAO)
- **[ETHPanda](https://ethpanda.org/)：** ETHPanda 是連線華語區與全球以太坊生態的非營利組織，定期整理和釋出優質的以太坊生態工作機會，同時提供 Bounty 任務和實習機會。ETHPanda 社群：[https://t.me/ETHPandaOrg](https://t.me/ETHPandaOrg)
- **專案方官方 Discord / Telegram 社群：** 許多 Web3 專案會在自己的官方社群中釋出招聘資訊，尤其是一些早期專案或需要社群參與的職位。
- **Twitter(X)：** 關注 Web3 領域的 KOL、專案方創始人、投資機構和招聘專家，他們經常會在 Twitter 上分享招聘資訊或行業洞察。
- **LinkedIn：** 雖然 Web3 領域相對新興，但許多成熟的區塊鏈公司和傳統企業轉型 Web3 的部門仍會在 LinkedIn 上釋出職位。同時，LinkedIn 也是瞭解專案方背景和人脈拓展的重要工具。
- **Web3 行業新聞和研究報告：** 關注 CoinDesk、The Block 等媒體，以及 Messari、Nansen 等研究機構的報告，從中可以發現新興專案和潛在的就業機會。
- **Web3 線下活動/線上 AMA：** 參加行業會議、Meetup 或線上 AMA（Ask Me Anything）活動，是直接接觸專案方、瞭解招聘需求和進行人脈拓展的絕佳機會。

## 四、如何判斷專案方是否可靠？

在招聘平臺上釋出工作資訊的專案方可能良莠不齊。那怎樣選擇一份優質的崗位？如何判斷該專案方是否可靠呢？通常可以根據以下內容進行一個簡單的打分（滿分 120 分），幫助你進行初步篩選：

| 序號 | 評估項                                                                                      | 分值    |
| ---- | ------------------------------------------------------------------------------------------- | ------- |
| 1    | 專案方願景（專案方要做什麼，怎樣實現）                                                      | 0-10 分 |
| 2    | 專案方聲譽（是否在公共媒體上存在過多的負面輿情）                                            | 0-10 分 |
| 3    | 專案方創始人背景以及權力結構（可以在 https://cn.rootdata.com/people 中進行查詢）            | 0-10 分 |
| 4    | 專案方是否上市以及上市交易所層次                                                            | 0-10 分 |
| 5    | 專案方的辦公地點（美國/新加坡/日本/中國香港還是不明？）                                     | 0-10 分 |
| 6    | 專案方能否在領英上被檢索                                                                    | 0-10 分 |
| 7    | 專案方的總市值排名                                                                          | 0-10 分 |
| 8    | 專案方的創始年份                                                                            | 0-10 分 |
| 9    | 專案方的合規性（是否在 Coinbase 上市 / 是否擁有牌照 / 是否是龐氏騙局的變種）                | 0-10 分 |
| 10   | 專案方發言人的立場 / 觀點 / 價值觀是否與你匹配                                              | 0-10 分 |
| 11   | 專案方獲得了哪些投資機構的支援                                                              | 0-10 分 |
| 12   | 專案方的社群運營情況（是否有官方推特 / Discord 等，更新頻率如何，多少人關注，是否積極互動） | 0-10 分 |

![專案可靠性評估表](../images/job-platform/project-reliability-assessment.png)

::: info **評估建議**

- 每一條的評分標準為 0-10 分，在你的認知範圍內對這個專案進行打分。也可以透過 AI 輔助收集資料。
- 若涉及代幣薪酬，需問清 vesting/cliff、解鎖頻率與流動性，可參考 <https://academy.binance.com/zh/articles/token-vesting> 或 <https://www.coinbase.com/learn/crypto-basics/what-is-vesting>。
- 如果總分沒有達到及格線（例如，低於 72 分），那你可能就要多加小心。特別是如果你發現你想要投遞的專案僅僅是套一個殼（例如 AI 概念），並且業務模式以拉人頭為主，並沒有實際落地業務（例如藉助 Telegram 小程式發展下線），且在領英、CoinGlass、非小號等資料庫中名不見經傳，甚至都沒有官方推特，那你可能就要警惕了，你可能並不是在找 Web3 工作，而是在給詐騙團伙**助紂為虐**。

但是，這並不意味著分數低的專案方一定不行，如該專案還在初創階段，**因此，如遇拿不定主意的專案，可以考慮在 LXDAO 或者 ETHPanda 社群中進行問詢**，大家都會很樂意給出一些指導建議和了解的行業內幕。對於 0 經驗的求職者來說，直接投遞優質專案可能會被拒，可以考慮從分數稍低但有潛力的專案開始積累經驗。也可以在社群裡尋找內推的機會。
:::

## 五、Web3 職業發展路徑

### 1. 運營向

::: steps

1. **大使計劃（Ambassador Program）**
   ::: card
   如果你有意從事運營向路線但卻沒有太多 Web3 經驗甚至 0 經驗，那麼各類大使計劃將是一個不錯的選擇。大使計劃的特點在於門檻低、靈活性強、涉獵範圍廣。但是辯證來看，大使的薪資待遇較低（通常是按勞分配，並可能會有淘汰機制）且可能無法接觸到實質性的核心業務，但是仍不失為一種從 0 起步為自己積累 Web3 經驗的好辦法。
   :::
   **大使任務主要包括如下幾個方面的工作內容：**

   1. **社群運維：** 通常會扮演 Telegram / Discord / WeChat 社群的客戶服務角色，及時解答社群的問題，維護社群秩序。
   2. **內容製作：** 通常包括製作涉及專案方的研究報告、Meme 圖、YouTube 影片，並在 X（原 Twitter）等社交媒體上進行釋出；Figma 海報製作、Canva 海報製作等。
   3. **活動協助：** 通常包括協助實習生或 BD（商務拓展）開展線下活動 / AMA / Meetup，並在必要時擔任 Speaker；同時可能負責活動資料統計與激勵發放。

   **目前正在招募大使的專案：**

   - **ZetaChain 大使計劃：** [https://x.com/ZetaChain_CH/status/1896723097476317243](https://x.com/ZetaChain_CH/status/1896723097476317243)
   - **其他新興 Layer 2 / DeFi / NFT 專案：** 許多新專案在發展初期都會推出大使計劃以擴大社群影響力，建議關注相關專案的官方公告。
     <br>

2. **實習生（Internship）**
   ::: card
   相比於大使計劃，實習生則算是正式的初窺門徑。在工作難度與強度上，實習生比大使要高很多，但是一般情況下來說仍少於傳統行業中的實習生，並且普遍是以遠端工作為主。同時薪資待遇是大使的數倍。
   :::
   **實習生主要包括如下幾個方面的工作內容：**

   1. **前期的大使工作：** 實習生通常需要掌握並執行大使層面的基礎工作，作為對社群和專案基礎的瞭解。
   2. **獨立設計及釋出 Telegram / Discord 的運維或營銷文案：** 能夠獨立策劃並撰寫社群公告、活動宣傳、營銷內容等。
   3. **獨立設計 AMA（Ask Me Anything）：** 能夠獨立策劃 AMA 活動的主題、流程、嘉賓邀請和問題收集等。
   4. **收集社群成員反饋並進行上報：** 負責整理和分析社群使用者的意見和建議，並及時向專案團隊反饋。
   5. **組織並管理大使：** 協助管理和協調大使團隊的工作，分配任務，提供指導。
   6. **稽核大使撰寫內容並提出修改意見：** 負責對大使產出的內容進行質量把控和潤色。
      <br>

3. **正式職工（Staff / Faculty）**
   ::: card
   正式職工在 Web3 運營領域承擔更核心、更具戰略性的職責，通常需要具備豐富的行業經驗和獨立解決問題的能力。
   :::
   **常見的運營類正式職位包括：**

- **社群經理（Community Manager）：** 負責社群的整體運營策略、使用者增長、活動策劃、危機公關等，是專案與社群之間的橋樑。
- **市場經理（Marketing Manager）：** 負責制定和執行市場推廣策略，包括品牌建設、內容營銷、社交媒體運營、合作拓展等。
- **商務拓展（Business Development，BD）：** 負責尋找和建立與外部專案、機構、KOL 的合作關係，推動專案生態發展。
- **產品運營（Product Operations）：** 關注產品上線後的使用者反饋、資料分析，最佳化產品體驗，提升使用者活躍度。
- **內容創作（Content Creator）：** 負責制定專案的內容方向和風格，確保所有對外輸出內容的一致性和高質量。

### 2. 技術向

::: card
不同於運營，在技術層面可能沒有所謂的大使計劃，更多的是一步到位直接從實習生開始幹起，或者透過參與開源專案、駭客松等方式積累經驗。Web3 技術崗位的專業性強，對技術棧要求較高。
:::
**常見的 Web3 技術崗位包括：**

![Web3 技術崗位分類](../images/job-platform/web3-tech-positions.png)

- **區塊鏈核心開發者（Blockchain Core Developer）：** 負責底層區塊鏈協議的設計、開發和維護，如共識機制、P2P 網路等。通常需要精通 Go、Rust、C++ 等語言。
- **智能合約工程師（Smart Contract Engineer）：** 負責在區塊鏈上編寫、測試和部署智能合約，如 DeFi 協議、NFT 合約等。主要使用 Solidity、Rust（for Solana/Polkadot）等語言。
- **Dapp 後端開發者（Dapp Backend Developer）：** 負責 Dapp 的後端服務開發，如資料索引、API 介面等。可能涉及 Node.js、Python、Go 等語言，以及 IPFS、The Graph 等技術。
- **Web3 安全審計師（Web3 Security Auditor）：** 負責對智能合約和區塊鏈專案進行安全審計，發現並修復潛在漏洞。需要深入理解區塊鏈安全原理和常見攻擊模式。
- **密碼學工程師（Cryptography Engineer）：** 專注於密碼學演算法的設計和實現，應用於零知識證明（ZKP）、同態加密等領域。
- **DevOps 工程師（DevOps Engineer）：** 負責 Web3 專案的部署、自動化、監控和運維，確保系統的穩定性和效率。

**技術新人如何積累經驗：**

- **參與開源專案：** 貢獻程式碼到 GitHub 上的 Web3 開源專案，是學習和展示能力的好方式。
- **參加駭客松（Hackathon）：** 在短時間內與團隊協作開發 Web3 專案，快速學習新知識並獲得實踐經驗。如果沒有任何經驗，推薦參加 LXDAO 的休閒駭客松，比較容易上手。
- **構建個人專案：** 從簡單的 Dapp 開始，逐步構建更復雜的專案，例如一個 DeFi 協議、一個 NFT 市場或一個鏈上游戲，並上傳到自己的 GitHub。
- **學習線上課程和教程：** 按照你的興趣和職業路徑，掌握 Solidity、Go、Rust 等核心程式語言，以及區塊鏈基礎知識和相關工具。推薦關注 LXDAO 殘酷共學。
- **閱讀白皮書和技術文件：** 深入理解不同區塊鏈協議和專案的技術細節。
